# Fault-Tolerant Quantum Compilation: T-Count Minimization via ZX Preconditioning and Phase-Polynomial Resynthesis

You are an autonomous research agent working on **Fault-Tolerant Quantum Circuit Compilation: T-Count Minimization**.

## Goal

Build an optimization pipeline that minimizes the T-count of Clifford+T circuits through a combination of ZX-calculus preconditioning, Hadamard-segmented phase-polynomial resynthesis, and local cleanup. Outperform PyZX's built-in optimizers (`full_reduce`, `full_optimize`) and approach or exceed state-of-the-art T-count results on standard benchmark families.

## The Challenge

In fault-tolerant quantum computing, T gates are the dominant cost driver because each requires a distilled or cultivated magic state — consuming substantial physical qubit overhead. Cliffords are comparatively cheap but not free. The primary optimization target is T-count, with T-depth, Hadamard count, and CNOT count as secondary objectives.

**Core insight:** Clifford+T circuits admit polynomial-size path-sum representations, and many compilation pipelines work by isolating Hadamard-free phase-polynomial fragments separated by Clifford/Hadamard boundaries. Within each fragment, the phase polynomial is:

```
f(x) = Σ_k (c_k / 8) · (x_{i1} ⊕ x_{i2} ⊕ ...)
```

Each term with an odd coefficient (c_k ∈ {1,3,5,7} mod 8) contributes a non-Clifford phase and counts toward T-count; even coefficients correspond to Clifford phases. Minimizing T-count becomes: **find an equivalent phase polynomial with fewer odd-coefficient terms** — a combinatorial optimization problem over Z_8 with rich algebraic structure.

**TEST_METRIC** = fraction of benchmarks where the pipeline produces a circuit with **strictly fewer T gates** than the best PyZX baseline (`full_optimize`), while passing equivalence verification. Every benchmark is counted in the denominator — errors, verification failures, and timeouts count as misses.

## Pipeline Architecture (Three Stages)

### Stage 1 — ZX-Calculus Preconditioning (structural reduction)

Apply ZX-calculus rewrite rules to simplify the circuit graph before algebraic optimization. PyZX's `full_reduce` applies spider fusion, local complementation, and pivoting in a fixed greedy order. Our contribution: **non-greedy search** over rewrite rule sequences.

- **Beam search** over alternative ZX simplification paths
- Use a **cheap proxy score** (non-Clifford phase count in ZX graph) to guide search
- After beam search completes, extract circuits from ALL beam top-k candidates plus the greedy `full_reduce` result. Compare actual T-counts across all extracted circuits and select the best.
- This is the **supporting module**, not the main novelty

Individual ZX rewrite rules available in PyZX:
- `spider_simp(g)` — fuse connected same-colored spiders
- `id_simp(g)` — remove identity spiders
- `lcomp_simp(g)` — local complementation
- `pivot_simp(g)` — pivot rule
- `bialg_simp(g)` — bialgebra rule
- `gadget_simp(g)` — gadget fusion / phase gadget simplification
- `pivot_gadget_simp(g)` — pivot on gadgets
- `pivot_boundary_simp(g)` — pivot on boundary vertices

### Stage 2 — Hadamard-Segmented Phase-Polynomial Resynthesis (core contribution)

Partition the ZX-simplified circuit into Hadamard-free phase-polynomial fragments separated by Clifford/Hadamard boundaries. Hadamard reduction and segmentation is itself important preprocessing — minimizing the number of Hadamard boundaries exposes larger phase-polynomial fragments amenable to global optimization.

Within each fragment, minimize odd-coefficient terms via:
- **Reed-Muller / TODD approach:** Term merging via linear algebra over Z_2 (PyZX has `todd_simp` built in — our job is to improve on it or combine it with other methods)
- **ILP/SAT formulation:** Encode optimal term-merging as an integer program. Use as an exact oracle on small/structured instances. (Exact T-count optimization is NP-hard, so scalable performance must come from relaxations and heuristics.)
- **Greedy heuristic resynthesis:** For fragments too large for exact methods

PyZX's `phase_block_optimize` already does Hadamard-segmented TODD optimization. Our pipeline must demonstrate improvement beyond this existing implementation.

### Stage 3 — Local Cleanup

- Merge adjacent rotation gates on the same axis
- Peephole-optimize small subcircuits (2-3 qubits)
- Optionally apply matroid partitioning as a T-depth reduction pass (targets parallelism, not T-count directly)
- Final `basic_optimization` pass for commutation/cancellation cleanup

## Equivalence Verification

**Every optimized circuit MUST be verified equivalent to the original.**

For circuits ≤ 6 qubits: use `circ.verify_equality(optimized)` (unitary comparison).
For circuits > 6 qubits: ZX-calculus identity test — compose original with adjoint of optimized, run `full_reduce`, and classify the result:

Verification returns one of three states:
- **"verified"**: No internal vertices remain after simplification — provably identity.
- **"failed"**: Internal vertices with **non-Clifford phases** survive `full_reduce`. The composition of equivalent circuits cannot produce non-Clifford residue after full simplification, so this is a definitive mismatch. Result MUST be discarded.
- **"unverified"**: Only Clifford-phase internal vertices remain. This is inconclusive — could be identity with Clifford bookkeeping, or could be a Clifford-level mismatch. Result is reported but excluded from win/tie/loss statistics.

Summary stats must show verified, unverified, and failed counts separately. Win/tie/loss denominators use ONLY verified results.

## Research Directions

1. **Stage ordering and interaction:** The current pipeline runs Stage 1 → Stage 2 → Stage 3. But ZX extraction can destroy the phase-polynomial structure that Stage 2 needs. Experiment with:
   - Reverse order: phase-poly optimization on the original circuit first, then ZX preconditioning
   - Parallel paths: run both orderings and take the best result
   - Use `phase_block_optimize` as the Stage 2 engine (it handles Hadamard segmentation internally)
   - Apply Stage 2 to the original circuit AND to the Stage 1 output, compare

2. **ZX beam search improvements:** The proxy score (non-Clifford phase count) correlates with but does not equal T-count. Try:
   - Different proxy scores: total vertex count, gadget count, edge count
   - Larger beam widths and depths (trade compute for quality)
   - Rule ordering heuristics (which rules tend to unlock further simplifications?)
   - Monte Carlo tree search over rewrite sequences
   - Simulated annealing over rule orderings

3. **Phase-polynomial optimization beyond TODD:** The core research contribution lives here.
   - **ILP/SAT formulation:** Encode the term-merging problem as an integer program. For each pair of odd-coefficient terms, can they be merged (summed to an even coefficient) via Z_2-linear recombination? Use PySAT or OR-Tools. Exact on small fragments, relaxations at scale.
   - **Greedy merge heuristics:** Different priority orderings for term merging — merge highest-weight terms first? Merge terms with most shared variables? Try multiple random orderings and keep the best.
   - **Hadamard reduction:** Fewer Hadamard boundaries → larger phase-polynomial fragments → more optimization opportunity. Implement Hadamard minimization as explicit preprocessing.
   - **Reed-Muller structure exploitation:** Phase polynomials over Z_2 have structure from Reed-Muller codes. Exploit this for more efficient merging.
   - **Hybrid exact/heuristic:** Use ILP on small fragments (≤8 T gates), greedy on larger ones.

4. **Post-processing and cleanup:**
   - Peephole optimization: exhaustively search over short Clifford+T sequences for 2–3 qubit subcircuits
   - T-gate commutation: push T gates through Cliffords to create new cancellation opportunities
   - Matroid partitioning for T-depth reduction (secondary metric)

5. **Combining with Qiskit/external tools:** Import benchmark circuits from established benchmark suites (e.g., RevLib, QASMBench) if they come with verified Clifford+T decompositions.

6. **Updating stats and notes:** Continuously update NOTES.md and progress plots with findings after every iteration.
