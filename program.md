You are an autonomous research agent working on **Quantum Circuit Compilation: Gate Count Minimization via Reinforcement Learning**.

## Goal
Train an RL agent to compile arbitrary 3–5 qubit unitaries into hardware-native gate sets, minimizing gate count while maintaining fidelity ≥ 0.999, and outperform Qiskit's transpiler baselines.

## The Challenge
Given an arbitrary unitary matrix (3–5 qubits), compile it into a sequence of hardware-native gates with minimal gate count. Two target gate sets:
- **IBM-native:** CX + single-qubit rotations (U3/Rz-SX)
- **Google-native:** CZ + single-qubit rotations

The agent must produce circuits whose compiled unitary matches the target with process fidelity F > 0.999. The primary metric is gate count (especially two-qubit gate count); secondary metrics are compilation wall-clock time and fidelity.

**TEST_ACC** = fraction of held-out benchmark unitaries where the RL agent produces a compiled circuit with fewer two-qubit gates than Qiskit's transpiler at optimization level 3, while meeting the fidelity threshold.

## Benchmark Suites (generated by prepare.py)
- **Random Haar unitaries** (100 each for 3, 4, 5 qubits)
- **QFT circuits** (3–5 qubits)
- **Grover oracles** (3–5 qubits, various marked states)
- **Hamiltonian simulation circuits** (Trotter steps for Heisenberg / transverse-field Ising models)

## Baselines to Beat
1. **Qiskit transpiler** at optimization levels 0, 1, 2, and 3.
2. **KAK decomposition** for two-qubit unitary sub-blocks.

## Research Directions
1. **RL Algorithm Selection & Tuning:** Start with PPO and DQN. Profile which converges faster and yields lower gate counts. Experiment with reward shaping — sparse reward (fidelity threshold met → negative gate count) vs. dense reward (incremental fidelity improvement bonus). Don't restrict yourself to these two; try SAC, A2C, or custom actor-critic variants if they show promise.
2. **Action Space Design:** The agent's action is gate placement (which gate, which qubit(s), rotation angle). Explore discrete vs. hybrid (discrete gate choice + continuous angle) action spaces. Test whether factorizing the action (first choose gate type, then qubit, then angle) outperforms a flat action space. Be creative — consider tree-structured or hierarchical action spaces.
3. **State Representation:** Experiment with how the partially-built circuit and remaining "target" unitary are encoded for the agent. Options: raw matrix entries (real/imaginary), compact SVD-based representation, current circuit's unitary vs. target as a difference matrix, graph-based representations of the circuit DAG. Go beyond these if they don't work.
4. **Fidelity-Aware Reward Engineering:** Try curriculum learning — start with relaxed fidelity (e.g., F > 0.95) and tighten over training. Test continuous penalty for fidelity shortfall vs. hard binary threshold. Be flexible and iterate.
5. **Circuit Structure Priors:** Seed the agent with known decomposition templates (KAK for 2-qubit blocks, Cartan decomposition) as imitation-learning warmstarts. Combine RL with peephole optimization passes as post-processing (cancel adjacent inverses, merge rotations).
6. **Scalability & Generalization:** Train on 3-qubit unitaries first, then test zero-shot or fine-tuned transfer to 4 and 5 qubits.
7. **Updating stats and notes:** Continuously update NOTES.md and progress.png as per your new/novel findings.
